v5.2.8-FARM-CHECKPOINT｜Farm Sidepath Checkpoint

Ray's CPBL Data Site 在 v5.2 正式完成二軍旁路系統第一階段。這不是封版，而是二軍大坑開挖前的穩定檢查點： 二軍爬蟲、二軍賽程頁、二軍比賽中心、首頁入口、網站介紹與版本紀錄都已經串起來。 目前二軍資料仍走獨立 farm 資料流，爬蟲只寫入 data/farm/farm-schedule-2026.json， farm-schedule.html 可依月份、球隊、場地與狀態篩選，並可點入 farm-match.html 二軍比賽中心。 二軍比賽中心採用一軍 Match Center 的視覺樣式，但仍只讀二軍 schedule raw，不使用假逐局、假打者或假投手表。 一軍主流程、live-boxscore.json、PREGAME / LIVE / FINAL fetch 都維持穩定不動，下一階段將進入 v5.3 二軍 boxscore 旁路大坑。

v5.1.4-SIDELOAD-DASHBOARD-OVERRIDE｜Control Panel Era

Control Panel
  • 新增 teams-dashboard.html 球隊資料總覽 Dashboard，以旁路頁面讀取資料，不寫入、不污染主資料。
  • Dashboard 以 live-boxscore 作為主要資料中心，可整理六隊戰績、排名、近況、最近一場、下一場、資料完整度與本季對戰摘要。
  • 新增 admin-manual-override.html Manual Override 管理頁，先採 JSON 產生器模式，不直接寫檔。
  • Manual Override 產生器可建立 gameSno、日期、主客隊、狀態、R/H/E、勝敗救、MVP、finalLock 與修正原因等片段。
  • 新增 data/manual/manual-boxscore-overrides.json 作為人工修正資料入口，平常可保持空陣列，只有 CPBL 官方資料異常時才啟用。
  • FINAL fetch 支援 Manual Override Array 格式，人工修正可在最後一層套用，不改變原本 parser 與 recentFinalGuard 主流程。
  • LIVE fetch 加入 Manual Override 接線，可在 createOrMergeGame 後、寫入 live-boxscore 前套用安全修正。
  • 完成第 3 項與第 4 項旁路功能：球隊資料總覽 Dashboard 與 Manual Override 管理頁。
